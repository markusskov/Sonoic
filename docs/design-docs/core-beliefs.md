# Core Beliefs

Sonoic should make everyday Sonos control feel fast, legible, and dependable.

## Product Beliefs

- Local-first Sonos control is the center of the product.
- The active room or group matters more than broad browsing.
- A small honest control surface is better than a wide unreliable one.
- Stale or unavailable state should be visible instead of disguised as fresh state.
- Real household behavior should beat sample data and generic scaffolding.

## Engineering Beliefs

- Build one real vertical slice at a time.
- Keep the SwiftUI structure direct and feature-shaped.
- Let repeated concrete pressure create abstractions.
- Keep shared code reserved for behavior truly shared across targets.
- Prefer checkable repo-local knowledge over private memory or chat-only decisions.

## Agent Beliefs

- Agents need a map first, then focused context.
- Documentation should explain why the shape exists, not narrate every line of code.
- Execution plans are for risk, coordination, and memory. They are not ceremony for tiny changes.
- When an agent gets stuck, improve the harness, docs, or checks so the next run has a better path.
